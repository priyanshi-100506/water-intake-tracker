QUICK START - AI Hydration Intelligence Platform
================================================================================

🚀 FASTEST WAY TO GET STARTED (5 MINUTES)

STEP 1: Install Dependencies
------------------------------
pip install -r requirements.txt


STEP 2: Start Services (open 3 terminals)
---------------------------------------------

Terminal 1 - Start Ollama Server:
  ollama serve
  
Terminal 2 - Start API Server:
  python -m uvicorn src.api:app --host 127.0.0.1 --port 8000 --reload
  
Terminal 3 - Start Dashboard:
  streamlit run dashboard_new.py
  Then open: http://localhost:8501


STEP 3: Use the Dashboard
--------------------------
1. Click "New User" in sidebar
2. Fill in your details (age, weight, activity level, goal)
3. Click "Log & Analyze" to log water intake
4. View your analytics and AI-powered recommendations!

API ENDPOINTS (HTTP)
================================================================================

USER MANAGEMENT:
  POST   /profile                  Create user profile
  GET    /profile/{user_id}       Get user profile
  PUT    /profile/{user_id}       Update profile
  DELETE /user/{user_id}          Delete user

HYDRATION:
  POST   /goal                     Set daily goal
  POST   /log-intake              Log water intake (gets AI analysis!)
  GET    /history/{user_id}       View intake history

ANALYTICS:
  GET    /analytics/{user_id}     Get metrics (score, streaks, averages)
  GET    /weekly-report/{user_id} Get AI-powered weekly report

ACCESS DOCS:
  http://127.0.0.1:8000/docs     → Interactive API docs

================================================================================
TROUBLESHOOTING
================================================================================

❌ "Connection refused at 127.0.0.1:11434"
   → Ensure Ollama is running: ollama serve

❌ "Database is locked"
   → Kill Python processes, delete water_tracker.db, reinit

❌ "Model not found"
   → Pull model: ollama pull llama3.2

❌ "Port 8501 already in use"
   → streamlit run dashboard_new.py --server.port 8502

❌ "Slow responses"
   → Normal on first run (model loading). Subsequent requests are faster.
